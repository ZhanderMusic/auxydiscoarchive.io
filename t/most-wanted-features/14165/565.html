At the risk of banging on about possible ‘Pro+’ features…

Adding numbers to sound and fx control dials when editing might be a useful ‘pro’-level expectation.

e.g. when editing LPF or HPF the frequency value is visible. This would allow users to filter in/out frequencies more accurately and purposefully, rather than needing to do it by ear, while relying on the quality and characteristics of their speakers/headphones.

Another example might be showing Time for Offset.

It could/would also appear in the Automation screen, where applicable.

Numeric feedback is already available for controls which have extra dimensions - e.g. Reverb.
Ironically, we can currently make numeric frequency adjustments in the Reverb controls, but not on LPF/HPF.

(Just touching the control, without moving it, would also bring up the numeric value - similar to how recent generations of midi controller keyboards behave. Alternatively, maybe an ‘always on/when editing/off’ toggle in the app preferences?)

Side-thought.

Toggle between bespoke numeric values (e.g. 100% on Reverb Size) to a duration figure by tapping on the % unit. Or, again, possibly one for app preferences, for users who’d prefer less ambiguous values (most likely ‘pros’).

Auxy March 11, 2021, 8:52am #579

Technically, we can create (any number of) macro controls, and each control can control any number of parameters. That’s why we initially had the “vague” macros called “shape” and “character”. I.e. so that we could assign the knobs to different things depending on the type of sound. We have unified this for the most part though, because that approach has other benefits. Although, we still tune each control for each sound, e.g. setting the range for the filter.
