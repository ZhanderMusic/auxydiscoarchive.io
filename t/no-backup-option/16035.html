Just to clarify, like your original post says you have to import them one by one. There’s not a bulk import.

Imported projects get copied to the local app “sandbox” (each iOS app is sandboxes from others and can potentially have their own dynamic storage space).

If your projects had external samples they will not be included in the backups on iCloud. You’d have to track those down manually and import them fresh. Hope this helps some even if it’s not the answer you’re looking for.

Auxy July 19, 2019, 8:07am #12

When you import a sample, it scans the audio file and makes an identifier. Those identifiers are stored in the project. So when you open a project, it will look through the pool of samples and see if it can find a sample with the right identifiers. Ie this is regardless of path etc. So it should “just work” as long as you have the samples somewhere in your library.
