akabillposters May 11, 2018, 12:31pm #609
Fredrik:

Not sure I understand the question, but the next patch release (5.1.1) will fix the issues with some settings not being persisted (always chromatic

To clarify, the query was two-fold…

  1. Chromatic mode has historically been persistent within a project, so that when you enable it, all new loops/patterns in that project will default to Chromatic mode.
    This has been the behaviour since Auxy Studio.
    In 5.1.0, that stopped being persistent within a project – meaning even if enabled, new loops within the same project weren’t defaulting to Chromatic. i.e. it was broken.

  2. Historically, that setting has only been persistent within a project. Every new project defaults back to Minor (iirc).
    There has been a long-standing feature request to have that setting be persistent across all new projects.
    i.e. If I set it Chromatic within an open project, if I then start a new project, that setting will still be in effect – i.e. any new loops/patterns created will default to Chromatic.

Your use of the word ‘persistent’ confused me as to whether 5.1.1 would fix the break from 5.1.0 (#1 above) or additionally update it in line with the feature request (#2 above).

It’s still not clear, but I’m hopeful that 5.1.1 includes both #1 and #2 above.
