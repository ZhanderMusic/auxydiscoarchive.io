JPNYC January 30, 2018, 11:35pm #365

The reason for using .wav is technical. Wav files are uncompressed and compatible with both Windows and Mac. M4a is a compressed and lossy format.

What that means, is that every time you record, use or save a file in .m4a it looses some of the original quality (hence ‚Äòlossy‚Äô). That‚Äôs not however, the only reason to not use .m4a

When a lossy format (.mp3,MP4,m4a,h.264,h.265, .jpg etc.) are used in a project (any project, on any system) the hardware (computer, iPhone, iPad, etc.) has to decompress the file.

Files are compressed for many reasons, the biggest being size. A compressed file might be 5 megabytes, an uncompressed file might be 40 megabytes. So a compressed file is great for streaming (like music and movies) because it doesn‚Äôt take up a lot of space, doesn‚Äôt use a lot of data and sounds pretty close to the original file.

However, working with a compressed file when producing content, is not so good an idea, because you want to use the fullest quality possible in your tracks or movie. Also, like unpacking a box full of stuff, compressed files are squished down, so that they don‚Äôt take up much space, but have to be unpacked every time the are played or used.

This need for constant decompression when using lossy files, isn‚Äôt a big deal when streaming, but can put a huge strain on your DAW, video editing app, etc. causing you to use more processing power and battery.

.wav, .aiff, .tiff, .nef, .mov, raw, Apple lossless, .flac etc, are better when it comes to production, because they don‚Äôt need to be ‚Äòunpacked‚Äô (decompressed) and have the full quality of the orginal, as long as they are saved again in in an uncmpressed format such as .wav (not lossy, they lose no quality when you use them).

So this is why Auxy uses .wav. Why Ableton uses .wav or .aiff in samples and so many others.

It‚Äôs also why Auxy can export in .wav, to give you access to your full quality song, before it‚Äôs uploaded and streamed in a compressed format.
