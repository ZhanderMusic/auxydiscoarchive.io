OIL:

I’d love a melodic sampler.

Having recently done my first real attempt at using imported samples for melodic leads, I definitely think this is a ‘must-have’. Although the drum kit supports pitch automation, that UX/UI model doesn’t really hold up for true melodic use.

I appreciate that adding another way of doing things to the app isn’t ideal, but using the current drum kit model + pitch automation for melodic imports is a ‘kludge’/workaround that doesn’t scale well, especially when you have to create a new drum kit for each new melodic instrument anyway (because pitch automation works at drum kit level).

Even the language doesn’t scale – users shouldn’t use something called ‘drum kit’ to bring in samples for melodic use.

Maybe a ‘create instrument’ type of approach that allows you to import one or more sounds into a single ‘instrument’ (for layering/sound design purposes), with access to piano roll, and the basic instrument fx (LPF, HPF, delay, reverb, …) and automation for all those.

Imo, it’s something they should really look at revising/upgrading in a not-too-distant future update.

 
Who knows, maybe they already are.
